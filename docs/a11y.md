One violation was found and fixed during this audit: the "Save my progress on this
device" switch rendered `role="switch"` without `aria-checked` on web
(react-native-web does not map `accessibilityState.checked` for Pressable);
fixed with an explicit `aria-checked` prop in `app/src/components/SaveToggle.tsx`.

## Manual checklist

- **Contrast**: token ratios documented at the source (`app/tailwind.config.js` header):
  ink/bg 10.66:1, ink-muted/bg 5.71:1, ember-text 5.91:1, white/pine 7.5:1 — all ≥ 4.5:1.
  Raw `glow` gold is decorative fill only, never text.
- **Touch targets**: every Pressable carries `min-h-[48px]` (≥ 44px), verified by grep
  across `app/src/components/` and screens.
- **Screen-reader labels**: `accessibilityRole` / `accessibilityLabel` /
  `accessibilityState` on all interactive elements; headers marked
  `accessibilityRole="header"`; voice status and adversarial-test output announced
  via `accessibilityLiveRegion="polite"`.
- **Focus order**: follows DOM order top-to-bottom on all three screens (header
  controls → language → primary action → personas); no positive `tabIndex`, no
  focus traps. Verified by tabbing through in the axe run session.
- **Reduced motion**: pulse/skeleton animations gate on `useReducedMotion`
  (`app/src/theme/useReducedMotion.ts`) and fall back to static.
- **Not covered here**: native iOS VoiceOver walkthrough (web run only) — the same
  accessibility props feed UIKit, but a device pass hasn't been recorded.
